# C++ Primer 第2章まとめ（ツジムラさん用・コピペ版）

> 目的：第2章で出てきた「型・初期化・宣言/定義・const・参照/ポインタ・型推論」周りを、実務で事故らない観点で整理する。  
> 方針：**用語の定義 → なぜそうなるか（原理） → よくある罠 → 安全な書き方**の順でまとめる。

---

## 1. signed と unsigned を混在させない

### 原則
**signed と unsigned を同じ式で混在させない。**

### 理由（原理）
- C++では式の評価時に「より表現範囲が広い/変換規則により」**符号なし（unsigned）へ暗黙変換**されることがある。
- すると、意図せず「負の値」が巨大な正の値になり、**比較やループ条件が壊れる**。

### unsigned のオーバーフロー例
```cpp
unsigned int u = 0;
u = u - 1;
std::cout << u << std::endl;
```

出力（32bit unsigned の例）：
```
4294967295  // 2^32 - 1
```

理由：
- unsigned は負を表せないため、演算は **mod 2^N（Nはビット数）** で回り込む。

### 終わらない unsigned の for
```cpp
for (unsigned u = 10; u >= 0; --u) {
    std::cout << u << std::endl;
}
```

なぜ終わらない？
- `u` が 0 になった後、`--u` で最大値に回り込む
- `u >= 0` は **unsigned では常に真**（0未満が存在しない）

### 終わる unsigned の while
```cpp
unsigned u = 11;
while (u > 0) {
    --u;
    std::cout << u << std::endl;
}
```
`u > 0` なので、0になった瞬間に止まる。

### 安全な書き方
- カウントダウンは基本 **signed** を使う：
```cpp
for (int i = 10; i >= 0; --i) {}
```

- どうしても unsigned を使うなら、この形（イディオム）：
```cpp
for (unsigned i = 10; i-- > 0; ) {}
```
ポイント：`i-- > 0` は「比較してから減らす」ので 0 で止まる。

## 2. リテラル（literal）とは何か

### 定義
**ソースコードに直接書かれている値**。

例：
```cpp
10          // 整数リテラル
3.14        // 浮動小数点リテラル
'a'         // 文字リテラル
"Hello"     // 文字列リテラル
true        // 真偽値リテラル
nullptr     // nullポインタリテラル
```

### サフィックス（suffix）
リテラルの型を明示できる：
```cpp
10u     // unsigned int
10L     // long
10LL    // long long
10.0f   // float
```

### 進数表記
```cpp
010     // 8進数（=8）
0x10    // 16進数（=16）
0b1010  // 2進数（C++14以降, =10）
```

## 3. 初期化は代入ではない

### 例
```cpp
int a = 10;   // 初期化：生成と同時に値を入れる
a = 20;       // 代入：既にある a の値を書き換える
```

### 原理
- **初期化**：オブジェクトの生成時に必ず一度行われる
- **代入**：生成済みオブジェクトに対して、後から値を入れ直す

### 重要な帰結
- **参照は「別名」なので、あとから別のものに付け替えられない**
  → したがって **参照は必ず初期化が必要**。

## 4. 宣言・定義・extern

### 定義（definition）
実体を作る（メモリが確保される）：
```cpp
int globalValue = 10;  // 定義
```

### 宣言（declaration）
「どこかにある」と知らせるだけ（メモリ確保しない）：
```cpp
extern int globalValue;  // 宣言
```

### ルール（実務での基本）
- **定義は1回だけ**
- 他ファイルから使うなら **extern 宣言**を置く

例：

**main.cpp**
```cpp
int globalValue = 10;  // 定義（1回だけ）
```

**other.cpp**
```cpp
extern int globalValue;  // 宣言（参照）
```

## 5. 参照とポインタ（本質の違い）

| 観点 | 参照 `T&` | ポインタ `T*` |
|---|---|---|
| 正体 | **対象の別名** | **アドレスを持つオブジェクト** |
| `nullptr` | 基本できない | できる |
| 付け替え（再バインド） | 不可 | 可能 |
| 自身のアドレス | 「参照そのもの」の実体は概念的に持たない（`&ref` は参照先のアドレス） | `&p` でポインタ自身のアドレスが取れる |
| 主な用途 | 引数の受け渡し（コピー回避 / 呼び出し元更新） | 「無い」を表す / 切替 / バッファ・配列・低レイヤ |

---

## 6. 参照（Reference）

### 定義
**オブジェクトの別名**。

```cpp
int value = 0;
int &ref = value;
```

- `ref` は `value` の別名
- `ref` 経由の操作は `value` に直撃する

例：
```cpp
ref = 10;
std::cout << value << std::endl; // 10
std::cout << &ref << std::endl;  // value のアドレス（参照先のアドレス）
```

### 特徴
- **宣言時に必ず初期化**
- **再バインド不可**
- 「参照の参照」は作れない（文法上折りたたまれる）

### ポインタへの参照
「ポインタ変数そのものの別名」を作れる：
```cpp
int* p = &value;
int*& pref = p;   // p の別名
```

---

## 7. ポインタ（Pointer）

### 定義
**他オブジェクトのアドレスを持つオブジェクト**。

```cpp
int value = 0;
int* p = &value;   // value のアドレス
```

### 主要操作
```cpp
*p = 10;           // デリファレンス：指す先を操作
std::cout << *p;   // 指す先の値
std::cout << p;    // 指す先のアドレス
std::cout << &p;   // ポインタ自身のアドレス
```

### 特徴
- **オブジェクト**である（だから自分のアドレスが取れる）
- **再代入（指す先の変更）が可能**
- `nullptr` にできる
- ポインタのポインタも可能（`T**`）

---

## 8. 記号の意味整理

### 宣言時の記号
```cpp
int &ref = value;   // 参照
int *p = &value;    // ポインタ
int *&pref = p;     // ポインタへの参照
int **pp = &p;      // ポインタへのポインタ
```

### 式の中の記号
```cpp
&value   // value のアドレス
&ref     // 参照先（value）のアドレス
*p       // ポインタが指す先の「オブジェクト」
p        // ポインタが保持するアドレス値
&p       // ポインタ変数自身のアドレス
```

> 重要：`*p` は「値」ではなく「指している先の **オブジェクト**」。  
> だから `*p = 3;` のように左辺にも置ける（＝lvalue）。

## 9. const / constexpr / inline（ここは誤解が出やすいので訂正込み）

### const
- **初期化が必須**
- 以後、その変数を通して再代入できない
- 実行時初期化でもOK（「コンパイル時定数」とは別）
```cpp
int getNum();
const int v = getNum(); // OK（実行時に決まってもよい）
```

### constexpr（何の略？）
- **constant expression（定数式）** の略
- 「コンパイル時に評価できることを要求」する（要求が強い）
```cpp
constexpr int x = 10;  // OK
```
※ `constexpr int v = getNum();` は **getNum() が constexpr 関数**など、コンパイル時評価できる条件を満たす必要がある。

### inline（ここは訂正：プリプロセッサではない）
- `inline` は **プリプロセスで展開される命令ではない**
- 意味は主に2つ：
  1. 関数に対して：多重定義ルールに関する緩和（同じ定義を複数翻訳単位に置ける方向のルール）＋最適化のヒント  
  2. 変数に対して（C++17〜）：**ヘッダに置いても多重定義になりにくい**（ODR対策）

> 「必ずインライン展開して高速化」ではない。  
> インライン展開（関数呼び出しを展開するかどうか）は最終的に **コンパイラが最適化で決める**。

### inline constexpr を現代でよく使う理由（正しい説明）
- ヘッダに定数を置きたい（複数 .cpp から共通で使いたい）
- `inline constexpr` なら **ヘッダ定義が扱いやすい（ODR問題を避けやすい）**
- 速度面の話は「定数が式に埋め込まれるか」は主に最適化次第で、`constexpr` だけでも十分に埋め込まれることが多い  
  → `inline` の主目的は「リンク/多重定義のルール」側。

## 10. top-level const / low-level const（「落ちる/無視」の意味を明確化）

### 用語の定義
- **top-level const**：その変数“自体”が const  
- **low-level const**：参照/ポインタが指す先が const

### top-level const の例
```cpp
const int v = 10;     // v 自体が const
int* const p = &x;    // p 自体が const（指す先は変えられない）
```

#### 「無視される/落ちる」って何？
**コピー（別の変数を作る）**をするとき、top-level const はコピー先に引き継がれないことがある：
```cpp
const int ci = 10;
int x = ci;  // x は int（const が付かない）
```
理由：コピー先を変更しても、元の `ci` は変わらないので安全。

### low-level const の例
```cpp
const int* p = &v; // const int へのポインタ（*p は書き換え不可）
const int& r = v;  // const int への参照（r 経由で書き換え不可）
```

low-level const は外せない：
```cpp
const int ci = 10;
const int* p = &ci;
int* q = p;  // NG（もしOKなら *q=... で ci を壊せる）
```

## 11. 型別名（Type Alias）

### 目的
型が長いときに短くして、意図も伝える。

- 昔：`typedef`
- 今：`using`（読みやすい）

例：
```cpp
using Vec = std::vector<int>;
Vec a = {1,2,3};
```

### 罠：別名がポインタ型だと const が「ポインタ側」に付く
```cpp
using pstring = char*;
const pstring cstr = nullptr; // char* const（ポインタが const）
```
直感で `const char*` と誤読しやすいので注意。

## 12. auto（型推論）

### 定義
右辺の初期化式から型を推論する。
```cpp
auto x = 1;      // int
auto y = 3.14;   // double
```

### 訂正ポイント
- `auto` は「右の式がコンパイル時に評価される」ことを意味しない  
  → **型を推論するだけ**。式が実行時評価かどうかとは別問題。

### const / 参照との関係（ざっくり）
- `auto` は基本的に **参照性を落とし**、**top-level const を落とす**ことが多い
- 参照として受けたいなら `auto&` を書く

## 13. decltype（式の型をそのまま取る）

### 定義
`decltype(式)` は、その式の型を取得する（式は実行されない）。

### 重要な注意：lvalue なら参照型になることがある
```cpp
int x = 10;
int* p = &x;

decltype(x)  a = 1;  // int
decltype(*p) b = x;  // int&  ← ここが罠
```

#### なぜ `decltype(*p)` は `int&` になる？
- `*p` は「指す先の **オブジェクト**」を表す式
- これは代入の左辺に置ける（lvalue）
- `decltype(lvalue式)` は参照型（`T&`）になりやすい

※ `int` そのものが欲しいなら、値になる式にする：
```cpp
decltype(*p + 0) c = 0; // int（+0 で値にする）
```

---

## 14. 「*p は値ではなくオブジェクト」を腹落ちさせる補足

- `cout << *p` で値が出るので「*p は値」と思いやすいが、実は **オブジェクト式**。
- だから `*p = 123;` が書ける（値リテラルは左辺に置けない）。

## 16. 参照・ポインタの意義（まとめ）

### 参照が必要な理由
- **大きいデータのコピーを避ける**（`const T&`）
- **呼び出し元を変更する**（`T&`）

### ポインタが必要な理由
- `nullptr` で **「無い」を表せる**
- **指し先を後から切り替えられる**
- 配列・バッファ・低レイヤI/Oなど、**アドレス操作が本質の領域**で必須

---

## 付録：超短いチェックリスト（事故防止）
- unsigned で `i >= 0` みたいなループを書かない
- 参照は必ず初期化、付け替え不可
- `const` と `constexpr` を混同しない（実行時 const は普通にある）
- `inline` は「必ず展開して速くする魔法」ではない（主にODR/リンクの都合）
- `decltype((x))` は参照になりやすい（括弧で事故る）
- `using p = T*; const p` は「ポインタが const」
