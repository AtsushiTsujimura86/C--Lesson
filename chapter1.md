## C++の実行プロセス
1. ソースコード作成(.cpp)
2. プリプロセス(#include, #define、マクロ展開)
3. コンパイル（C++⇒アセンブリ言語）
4. アセンブル（アセンブリ言語⇒オブジェクトファイル(.o、機械語)）
5. リンク（オブジェクトファイルとライブラリを結合、実行ファイル）
6. 実行

## コンパイル（ビルドコマンド）
```bash
// 最小構成
g++ a.cpp -o a
// 厳密
g++ -std=c++20 -Wsall -Wexttra a.cpp -o a
```
・ｇ＋＋の説明
・プロパティの説明
・-oは実行ファイル名の指定、指定しない場合a.outになる
・実行プロセスのそれぞれの段階だけをすることもできる（g+＋プロパティ）
このコマンドで実行プロセスの2～5を実行している。

## Tips
### 終了コードの確認
```cpp
int main(){
  return 0;
}
```
このreturnはOSに０という終了コードを投げている。
```$ echo $?```で直前の終了コードを確認できる。`$?`はシェルスクリプトとかでもでてくる。

## 関数の参照渡し
「参照」というと参考にするみたいな感じで勘違いしがちだが、実際は「本体そのもの」を別名で渡しているという意味合い。
普通に引数として渡すのはただのコピーを渡しているだけ。コピーの内容を変えても元の変数に何ら変化はないが、参照を渡すと本体の内容を変えられる。
```
// 参照渡しでない関数
void func(int x){
  x = 100;
}
int a=10;
func(a);
std::cout << a << std::endl; //a=10のまま

// 参照渡しの関数
void func(int& x){
  x = 100;
}
int a=10;
func(a);
std::cout << a << std::endl; //a=100になる。
```


## 標準入力、出力
std::coutはostream型のオブジェクト、std::cinはistream型のオブジェクト。
### std::cout << 10 << 20 <<std::endlができる理由
<<の定義が```&ostream operator<<(ostream& os, int value)```という関数になっている。つまり、<<の左側にあるオペランドと右側にあるオペランドに対して処理を行い、ostreamのオブジェクトそのものを返す関数になっている。
```cout << 10 << 20```はまず```cout << 10```が処理され、その後その結果と20が演算される。```(cout << 10)  << 20```のように連鎖呼び出しの形式で実行される。なお、operator<<()関数の中では、バッファやコンソール出力の内部処理が行われる。

### while, ifの条件としてのcin
よくある形のせつめい
```cpp
int value;
if(std::cin >> value)
  // 1回目の処理
  while(std::cin >> value)
    std::cout << value << std::endl;
```
入力が1_2_3(_はスペース)の場合、if文の中のcinで１が拾われて1回目の処理が行われる。その後の2はwhileの中で拾われて、その中の処理が行われる。
if, whileでは、条件文がfalseになれば抜けるが、この場合はstd::cin >> valueがfalseを返すのは、「int以外が入力された」or「EOF」の場合である。
ちなみにEOFは「Ctrl + D」をコンソールに入力するとできる。
