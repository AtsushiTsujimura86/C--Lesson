## C++の実行プロセス

### 全体の流れ

1. ソースコード作成（`.cpp`）
2. プリプロセス（`#include`, `#define`, マクロ展開）
3. コンパイル（C++ → アセンブリ言語）
4. アセンブル（アセンブリ → オブジェクトファイル `.o`）
5. リンク（`.o` + ライブラリ → 実行ファイル）
6. 実行

---

### 各工程を個別に実行する方法

```bash
# プリプロセスのみ
g++ -E a.cpp

# コンパイル（アセンブリ生成）
g++ -S a.cpp

# アセンブル（.o生成）
g++ -c a.cpp

# リンク
g++ a.o -o a
```

---

## コンパイル（ビルドコマンド）

```bash
# 最小構成
g++ a.cpp -o a

# 推奨（警告付き・規格指定）
g++ -std=c++20 -Wall -Wextra a.cpp -o a
```

### g++とは
GNU C++コンパイラ。  
内部では以下のツールを呼び出している「司令塔」：

- `cc1plus`（コンパイル）
- `as`（アセンブル）
- `ld`（リンク）

---

### オプション説明

- `-std=c++20`  
  使用するC++規格を明示する（必ず指定するのが望ましい）

- `-Wall -Wextra`  
  多くの警告を表示する（バグの早期発見）

- `-o a`  
  出力実行ファイル名を指定  
  未指定の場合は `a.out`

---

### 本質

```bash
g++ a.cpp -o a
```

は

> プリプロセス〜リンク（工程2〜5）をまとめて実行している

## 終了コード

```cpp
int main(){
  return 0;
}
```

`return` は OS に終了ステータスを返している。

- `0` → 成功
- `0以外` → 失敗扱い

確認方法：

```bash
echo $?
```

※ Linuxの終了コードは 0〜255（8ビット）  
`return -1;` は実際には `255` になる。

## 関数の参照渡し

### 本質

参照 = 本体そのものの別名  
コピー渡し = 値の複製を渡す

---

### コピー渡し

```cpp
void func(int x){
  x = 100;
}

int a = 10;
func(a);
std::cout << a << std::endl; // 10のまま
```

---

### 参照渡し

```cpp
void func(int& x){
  x = 100;
}

int a = 10;
func(a);
std::cout << a << std::endl; // 100になる
```

参照は「参考」ではなく「同一実体への別名」。

## 標準入力・出力

- `std::cout` → `std::ostream` 型のオブジェクト
- `std::cin` → `std::istream` 型のオブジェクト

---

### `std::cout << 10 << 20` が可能な理由

`<<` は以下のような関数として定義されている：

```cpp
std::ostream& operator<<(std::ostream& os, int value);
```

#### ポイント

- 左オペランド：`ostream`
- 右オペランド：値
- 戻り値：`ostream` の参照

---

#### 連鎖の仕組み

```cpp
cout << 10 << 20;
```

は内部的に：

```cpp
operator<<( operator<<(cout, 10), 20 );
```

1回目で `cout` を返すため、続けて `<<` が使える。

## while / if の条件としての cin

```cpp
while (std::cin >> value)
```

### 仕組み

- `>>` は `std::istream&` を返す
- `istream` は `bool` に変換可能
- 入力成功 → `true`
- 失敗またはEOF → `false`

---

### 入力例

```
1 2 3
```

処理の流れ：

- 1 → 成功
- 2 → 成功
- 3 → 成功
- EOF → false → ループ終了

---

### EOFの入力方法

- Linux / Mac：`Ctrl + D`
- Windows：`Ctrl + Z`

## Chapter1の重要概念まとめ

1. C++は「型」を中心に設計されている
2. クラスは「型 + 振る舞い」
3. 演算子はオーバーロード可能
4. ストリームはオブジェクト
5. プログラムはOSに終了コードを返す

## ビルドとコンパイルの違い

- コンパイル = 翻訳（C++ → 機械語への途中工程）
- ビルド = 実行可能ファイル完成までの全工程
